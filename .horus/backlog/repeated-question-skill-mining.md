# repeated-question-skill-mining — repeatedly-asked questions are undeclared skills

## Why

Questions the owner asks across sessions ("what's our usage looking like",
"paste that formatted", "what changed since v X") are candidate skills or
conventions nobody has written down. Owner named the family; the agent
flagged it as the weakest draft of the 2026-07-20 set because the evidence
source is genuinely unresolved — full transcripts are deliberately not stored
in `.horus/`, so mining must work from session notes, receipts, memory files,
or a deliberately-kept instance log. Counter-evidence noted at shaping: two
repeated asks from the same day ("paste receipts in terminal", "consolidated
tables") were resolved via agent memory + skill-text feedback with no new
machinery — the mining loop may already exist informally.

## Intended outcome

Either a lightweight periodic ritual that turns a kept instance-list into
skill candidates the owner accepts, or an evidence-backed drop verdict
("agent memory + skill calibration already covers this") — both outcomes
close the card honestly.

## Broad boundaries

Explore-phase: gather instances first, build second (if at all). Non-goals:
no transcript storage in `.horus/`; no automatic skill generation; no
continuous monitoring.

## Open decisions for backlog-refine

- Where the instance list lives and who appends to it (owner note? memory
  file? a receipts section?).
- The threshold that turns instances into a skill proposal.
- Hold-at-shaping vs drop-now, given the informal loop already observed.
